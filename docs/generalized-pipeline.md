# Generalized New-Bill Pipeline — Seam Contracts

This document is the interface contract for the generalized-pipeline build. Every
workstream codes against these seams; changes to them are deliberate, documented
decisions, not drive-by edits. Companion schemas: `contracts/anchor_set.schema.json`,
`contracts/scenario.schema.json`. Interfaces: `apps/api/app/engine/lookup.py`,
`apps/api/app/case_store.py`.

## Thesis (unchanged)

The LLM is the mouth, not the brain. LLMs appear in exactly two places: vision
extraction of uploaded bill/EOB PDFs into strict schemas, and the voice agent's
conversational wording. Everything else — reconciliation, flag detection, all
database lookups, band computation, Medicare multiples, answer keys, scenario
generation — is plain, deterministic, testable code. Every number a user sees or
the agent speaks traces to a database query or a config value (provenance travels
with the number).

## The seven stages, as code seams

| # | Stage | Module | Input → Output |
|---|-------|--------|----------------|
| 1 | Extraction | `routers/documents.py` (vision, exists) | PDF → bill object / EOB object (separate; keyed CPT/HCPCS/DRG) |
| 2 | Reconciliation | `engine/reconcile.py` (new) + `engine/flags.py` (extended) | bill + EOB → matched lines + red_flags[] |
| 3 | Chargemaster lookup | `engine/lookup.py` → backend | (hospital, code) → gross_charge, cash_price, all payer rows |
| 4 | Plan-specific rate | `engine/lookup.py` | (hospital, code, payer, plan) → negotiated rate |
| 5 | Cross-payer band | `engine/lookup.py` | (hospital, code) → {p25, median, p75, min, max, n_payers} |
| 6 | External anchors | `engine/lookup.py` (medicare) + config | code → Medicare rate (professional/facility/global), RAND-norm estimate |
| 7 | The ask | `engine/dossier.py` (extended) + `engine/anchors.py` (new) | lines + anchors → per-line multiples, fair band, excess, ask |

## Lookup layer (`apps/api/app/engine/lookup.py`)

`BenchmarkLookup` protocol; three backends selected by `BENCHMARK_SOURCE` env:
- `fixture` (default) — wraps the 5-CPT seed exactly as today. Zero external
  services; keeps offline boot + existing tests green.
- `sqlite` — local SQLite chargemaster + medicare_rates (tests, offline demo).
- `supabase` — live Postgres (runtime; decision #7).

All three return identical shapes. In-process caching per (hospital, code) —
chargemaster data is stable context (cache it), case data is fresh (never cache).

## Anchor sets (`contracts/anchor_set.schema.json`)

Per bill line, an **anchor set**: every applicable benchmark with provenance.
Methods: `medicare`, `plan_rate`, `cross_payer_band`, `cash_price`,
`gross_charge`, `rand_norm_estimate`. Every anchor carries `source`,
`source_url`, `formula`, `confidence` (`high|medium|estimated`), and a display
`label`. Per-line derived fields: `medicare_multiple`, `fair_band`
(config multiples × Medicare), `rand_flag` (billed > rand_ceiling_multiple ×
Medicare), `excess_above_band`, `coverage` status.

Coverage statuses (decision #12): `full`, `thin` (in chargemaster, <3 payers),
`absent_from_chargemaster` (facility line, standard code, not in MRF → low/med
severity supporting flag), `professional_excluded` (separate billing entity —
NEVER flagged as absent).

## Case store (`apps/api/app/case_store.py`)

In-memory, case_id-keyed store for job_spec / dossier / benchmark_report /
allowed_numbers, with the Maya fixture as automatic fallback for `DEMO_CASE_ID`.
Kills the `deepcopy(DEMO_JOB_SPEC)` merge. Supabase persistence is additive
(write-through when configured), never required for boot.

## Scenarios (`contracts/scenario.schema.json`, `data/scenarios/`)

Each scenario: `scenario.json` (meta + coverage + provider entities), `bill.json`,
`eob.json` (null for self-pay), `answer_key.json`, plus rendered `bill.pdf`/
`eob.pdf`. Answer keys are **generated by code** (`data/scenarios/generate.py`)
querying the same lookup layer; tests run the pipeline on the artifacts and
assert exact equality with the answer key. Line items carry `billing_entity`
(facility vs professional) so the absent-from-chargemaster flag can gate on it.

## Flag taxonomy (Visual Brief §06, verbatim source: audit/brief-decoded/visual-brief-data.js)

duplicate, upcode, unbundle (NCCI PTP), phantom, markup (supply/drug),
nsa_balance_billing, eob_mismatch (line-level), denial, units_error,
absent_from_chargemaster. Thresholds/knobs in `config/verticals/medical_bills.yaml`
— config-not-code boundary holds.

## Verification bar (per approved plan)

Answer-key exact-match tests per scenario; existing tests + offline boot green;
`transform --check` extended; Playwright e2e before merge; per-case honesty audit
green; Maya's locked numbers reproduce through the generalized flow.
